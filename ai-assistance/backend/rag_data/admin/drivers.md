# Drivers

## What This Section Is

The Drivers section is where the admin verifies drivers, reviews uploaded documents, checks ratings, and monitors driver activity.

## Main Purpose

It helps the team decide who is approved to drive on the platform.

## What the Admin Sees

- Driver list
- Driver profile details
- Document review status
- Verification state
- Vehicle approval state
- Driver ratings and performance-related data
- Live driver monitoring

## Step-by-Step Actions

### 1. Open Drivers
- The admin clicks the Drivers section.
- The list of drivers appears.

### 2. Review driver profile
- The admin opens a specific driver record.
- The profile shows identity and verification information.

### 3. Review documents
- The admin checks uploaded national ID, license, and libre documents.
- The admin can approve or reject each document based on the evidence.

### 4. Verify or reject the driver
- Once documents are complete and valid, the admin can verify the driver.
- If documents are not acceptable, the admin can reject them and give a reason.

### 5. Check ratings
- The admin can review driver ratings and performance indicators.
- This helps with quality control and support decisions.

### 6. Monitor live drivers
- The admin can open live driver monitoring if allowed.
- This helps track current availability and movement.

## Important Rules For Bemnet

- Driver verification is a staff/admin workflow, not a driver self-action.
- Always explain why a driver may be pending, approved, or rejected.
- If the user asks about someone else’s private documents, use the scope and privacy rule from identity.
- If access is restricted, direct the user to the correct staff role or support.
