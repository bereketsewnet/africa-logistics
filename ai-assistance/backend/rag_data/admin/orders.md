# Orders

## What This Section Is

The Orders section is where the admin manages orders, assignments, live dispatch, guest orders, and order status changes.

## Main Purpose

It gives staff the power to monitor and correct the delivery workflow.

## What the Admin Sees

- Order list
- Order status
- Live driver assignment
- Guest orders
- Order chat
- Dispatch tools
- Order stats
- Order detail view
- Cross-border order tools when applicable

## Step-by-Step Actions

### 1. Open Orders
- The admin opens the Orders section.
- The order list appears.

### 2. Filter or search orders
- The admin can search and filter by status.
- Guest orders and regular orders may be separated in different tabs or views.

### 3. Open an order detail
- The admin opens a specific order.
- The detail view shows route, status, driver, notes, and financial data.

### 4. Assign a driver
- The admin can assign a driver to a waiting order when allowed.
- This is part of dispatch and scheduling.

### 5. Update order status
- The admin can correct or advance an order status if their permission allows it.
- This is usually done for support, dispatch, or recovery cases.

### 6. Edit order details
- The admin can update order notes or details when needed.
- This helps correct mistakes or update operational information.

### 7. Cancel an order
- The admin can cancel an order according to system rules and permissions.
- This is different from a shipper canceling their own order.

### 8. Monitor live drivers
- The admin can open live driver dispatch tools.
- This helps match active drivers to orders.

### 9. Review guest orders
- Guest orders are handled in their own admin view.
- The admin can review and manage them separately from logged-in shipper orders.

## Important Rules For Bemnet

- Explain that order management is not the same as the shipper order page.
- Do not mix admin dispatch with shipper actions.
- If the user asks how to place an order, direct them to shipper docs unless they are specifically an admin placing an order on behalf of a user.
- If the user asks about live dispatch, explain it as a staff tool with permission checks.
