# Security Events

## What This Section Is

The Security Events section is the audit log for sensitive admin and system actions.

## Main Purpose

It helps the team investigate access attempts, permission failures, and important security-related events.

## What the Admin Sees

- Security event list
- Event type
- User and role information
- IP address
- Method and endpoint
- Reason field
- Metadata

## Step-by-Step Actions

### 1. Open Security Events
- The admin opens the security log section.
- The event list appears.

### 2. Review an event
- The admin opens an event record.
- They can inspect the reason and metadata.

### 3. Investigate access issues
- The log may show denied access attempts or permission failures.
- This helps diagnose why a staff member could not open a page.

## Important Rules For Bemnet

- Security logs are restricted and sensitive.
- Do not expose more than the user is allowed to see.
- If the user is asking about a denied action, explain that permissions are enforced by role.
- If the user needs access, direct them to the system owner or super admin.
