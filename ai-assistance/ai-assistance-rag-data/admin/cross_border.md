# Cross-Border

## What This Section Is

The Cross-Border section is where the admin manages international shipment workflows and customs-related reviews.

## Main Purpose

It helps staff inspect documents, border information, and cross-border order status.

## What the Admin Sees

- Cross-border order list
- Customs documents
- Border reference fields
- HS code and shipper TIN details
- Document approval or rejection controls
- ESW submission tools when enabled

## Step-by-Step Actions

### 1. Open Cross-Border
- The admin opens the cross-border section.
- Cross-border shipments appear.

### 2. Review documents
- The admin opens the attached customs or checkpoint documents.
- They can inspect the uploaded file and notes.

### 3. Approve or reject documents
- The admin reviews document correctness.
- They approve valid documents or reject invalid ones with notes.

### 4. Update border info
- The admin can update border information if allowed.
- This may include border references or related customs details.

### 5. Submit to ESW
- If the workflow is enabled, the admin can submit cross-border data to ESW.
- This is a backend integration step for customs workflows.

## Important Rules For Bemnet

- Cross-border workflows are sensitive and data-heavy.
- Do not mix them with normal domestic orders.
- If a user asks why a document is pending, explain that it is waiting for admin review.
- If the user asks about customs-specific action they cannot access, direct them to support or the correct staff role.
