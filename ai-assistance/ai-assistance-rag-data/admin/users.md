# Users

## What This Section Is

The Users section is where the admin manages shippers, drivers, staff, and general user accounts.

## Main Purpose

It lets staff search, review, activate, suspend, and update user records.

## What the Admin Sees

- User list with role and status
- Search and filtering
- Shipper users
- Driver users
- Staff users
- Account activation controls
- Staff creation controls

## Step-by-Step Actions

### 1. Open Users
- The admin opens the Users-related page or section.
- The list of users loads.

### 2. Search or filter users
- The admin can search by name, phone, email, or role depending on the section.
- Filters help separate drivers, shippers, and staff.

### 3. Open a user record
- The admin clicks a user row or card.
- The system shows the user details.

### 4. Activate or suspend a user
- The admin can toggle the active state if their permission allows it.
- This controls whether the user can continue using the platform.

### 5. Create staff accounts
- The admin can create new staff accounts when the permission is available.
- Staff accounts are for internal operations, not public users.

## Important Rules For Bemnet

- Be clear that user management is permission-controlled.
- Do not tell a driver or shipper that they can use these tools.
- If the user asks about user privacy or masked fields, explain that some staff roles receive redacted data.
- If the user lacks permission, tell them to contact the super admin or system owner.
