# Payments

## What This Section Is

The Payments section is where the admin reviews manual payments and withdrawal requests.

## Main Purpose

It lets staff approve or reject pending money-related actions safely.

## What the Admin Sees

- Manual payment review queue
- Pending payment count
- Withdrawal request queue
- Withdrawal pending count
- Approved amount and commission rate controls
- Proof images and admin review notes
- Payment/withdrawal history details

## Step-by-Step Actions

### 1. Open Payment Reviews
- The admin opens the Payment Reviews section.
- Pending manual payments and withdrawal requests appear.

### 2. Review manual payments
- The admin opens a payment entry.
- They inspect the payment proof and the payment details.
- They approve or reject the request according to the evidence.

### 3. Review withdrawal requests
- The admin opens a withdrawal request.
- They see the amount requested, bank details, user note, and proof image if attached.
- They may also see the user phone and email when the role allows it.

### 4. Set the commission rate
- For driver withdrawal approvals, the admin can set the platform fee percentage per request.
- The value can be adjusted instead of using only one global rate.

### 5. Approve the withdrawal
- The admin enters the approved amount and fee details.
- The system creates the correct wallet transaction.
- The platform fee now credits the admin wallet instead of reducing it.

### 6. Reject the withdrawal
- The admin can reject the request if the proof or details are not correct.
- A reason or admin note may be required depending on the workflow.

## Important Rules For Bemnet

- Payments and withdrawals are sensitive financial workflows.
- Always explain approval vs rejection clearly.
- Do not tell the admin to use driver-facing wallet steps.
- If the user asks why a commission appears in the admin wallet, explain that it is credited as revenue.
- If the user lacks payment permission, tell them to contact the correct staff role.
