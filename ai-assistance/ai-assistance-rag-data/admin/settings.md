# Settings

## What This Section Is

The Settings hub is where the admin manages platform configuration.

## Main Purpose

It centralizes system behavior, app settings, reference data, and feature toggles.

## What the Admin Sees

- System configuration
- Notification settings
- Vehicle types
- Countries
- Maintenance mode
- Pricing settings when linked
- Cargo settings when linked

## Step-by-Step Actions

### 1. Open Settings
- The admin opens the Settings hub.
- A group of configuration tools appears.

### 2. Update notification settings
- The admin changes which alerts are enabled.
- This affects platform-wide or staff-visible messaging behavior.

### 3. Manage vehicle types
- The admin adds or updates vehicle types.
- These values are used by pricing and order placement.

### 4. Manage countries
- The admin adds or updates countries.
- These settings are used for cross-border validation and location rules.

### 5. Update system config
- The admin changes system-wide configuration values.
- This can affect platform behavior such as pricing or operational toggles.

### 6. Use maintenance mode when needed
- The admin can put the platform into maintenance mode if the setting is available.
- This is for controlled downtime or system maintenance.

## Important Rules For Bemnet

- Explain settings as system-wide controls, not user profile settings.
- Do not tell a driver or shipper to change these values.
- If the user asks about a hidden setting, say it may depend on permission or a super-admin role.
