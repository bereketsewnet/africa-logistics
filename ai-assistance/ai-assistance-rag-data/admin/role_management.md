# Role Management

## What This Section Is

Role management controls which admin or staff roles can access each admin function.

## Main Purpose

It lets the system owner define permissions and staff capabilities safely.

## What the Admin Sees

- Current role permissions
- Staff role list
- Custom role creation
- Permission toggles
- Role deletion when allowed
- Personal permission summary

## Step-by-Step Actions

### 1. Open Role Management
- The admin opens the role management section.
- The list of roles and permissions appears.

### 2. Review a role
- The admin inspects the current permissions for a role.
- This shows what the role can and cannot do.

### 3. Update permissions
- The admin toggles permissions for a role.
- Changes affect access control across the admin dashboard.

### 4. Create or delete roles
- If allowed, the admin can create a new role or remove an unused one.
- This should only be done carefully.

### 5. Check personal permissions
- The admin can view what permissions their own account has.
- This is useful for troubleshooting access problems.

## Important Rules For Bemnet

- Role management is permission-sensitive.
- Do not suggest that a normal staff user can edit all roles.
- If a feature is blocked, explain that it depends on the role permission map.
- If the user asks for permission changes, tell them the system owner or super admin must handle it.
