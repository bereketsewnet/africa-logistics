# Report

## What This Page Is

The Report page is the driver's personal performance dashboard.

It gives the driver a clear summary of work, earnings, distance, rating, documents, and recent activity.

## Main Purpose

This page helps the driver understand how they are performing over a selected date range.

## What the Driver Sees

- Date range filters
- Quick presets like 7 days, 30 days, 90 days, and 1 year
- Report generation time
- Driver status and verification summary
- KPI cards
- Charts for daily jobs and earnings
- Job status breakdown
- Rating distribution
- Driver snapshot
- Recent jobs table
- Recent shipper feedback
- Document review history

## Step-by-Step Actions

### 1. Open Report
- The driver clicks **Report** in the sidebar.
- The report page loads the default date range.

### 2. Adjust date range
- The driver chooses a start date and end date.
- They can also use a preset range button.
- Clicking Apply refreshes the report.

### 3. Read the top summary
The top banner shows:
- Driver name
- Verification state
- Vehicle state
- Report period
- Generated time

### 4. Understand KPI cards
The KPI cards show the most important numbers, such as:
- Total jobs
- Active jobs
- Period earnings
- Total distance
- Average rating
- Cross-border jobs

### 5. Read the charts
- Daily chart shows jobs, completions, and earnings over time.
- Status chart shows how jobs are distributed by status.
- Rating chart shows ratings from 5 stars down to 1 star.

### 6. Inspect the driver snapshot
This area shows:
- Phone
- Email
- On-time rate
- Current streak
- Last trip date
- Document status
- Vehicle status

### 7. Review recent jobs
- The table shows the latest jobs in the selected date range.
- It includes reference, status, pickup, delivery, distance, earnings, and created date.

### 8. Review feedback and document reviews
- The feedback card shows recent shipper comments.
- The document review card shows approval or rejection history for submitted documents.

## Important Rules For Bemnet

- Always explain that the report is a summary of the driver's own work.
- Do not treat the report page like an admin analytics dashboard.
- If the user asks for company-wide data, redirect to admin or staff support.
- If data is empty, explain that it may mean no activity in the selected range.
- If the user needs a deeper explanation of a metric, define it in simple terms.
