# History

## What This Page Is

The History page is the shipper's transaction ledger.

It lists wallet movements such as credits, debits, commissions, tips, refunds, bonuses, and adjustments.

## Main Purpose

It helps the shipper understand how their wallet changed over time and lets them download a receipt for each transaction.

## What the Shipper Sees

- Search box
- Filter dropdown
- Transaction cards
- Type icon
- Amount with sign
- Transaction date
- Description
- Reference code if available
- Pagination controls
- Receipt download action

## Step-by-Step Actions

### 1. Open History
- The shipper clicks **History** in the sidebar.
- The transaction history page loads the latest wallet movements.

### 2. Search transactions
- The shipper types a keyword.
- The list filters by description or reference code.

### 3. Filter by type
- The shipper can filter by transaction category.
- This helps separate credits, debits, bonuses, tips, and refunds.

### 4. Move between pages
- The page shows a fixed number of records per page.
- The shipper can move to older or newer pages using the pagination buttons.

### 5. Read each row
- Credits and positive entries should be explained as money added.
- Debits and negative entries should be explained as money removed.
- The label and color help the shipper recognize the transaction type.

### 6. Download a receipt
- The shipper can download a PDF receipt for a transaction.
- The receipt contains the transaction ID, date, type, status, amount, description, and reference.

## Important Rules For Bemnet

- Use simple language to explain money movements.
- If the user asks why something was deducted, connect it to the transaction type or the wallet settlement flow.
- If the transaction is not visible, suggest using search, changing the filter, or checking the next page.
- Do not invent missing financial details.
- For balance disputes, direct the user to support or staff review.
