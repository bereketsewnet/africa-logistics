# Report

## What This Page Is

The Report page is the shipper's personal performance dashboard.

It gives the shipper a clear summary of orders, spend, distance, status mix, payment mix, and recent activity.

## Main Purpose

This page helps the shipper understand how they are using the platform over a selected date range.

## What the Shipper Sees

- Date range filters
- Quick presets like 7 days, 30 days, 90 days, and 1 year
- Report generation time
- Shipper name summary
- KPI cards
- Charts for daily orders and spend
- Order status breakdown
- Payment status breakdown
- Vehicle type breakdown
- Top routes
- Recent orders table
- Driver feedback summary

## Step-by-Step Actions

### 1. Open Report
- The shipper clicks **Report** in the sidebar.
- The report page loads the default date range.

### 2. Adjust date range
- The shipper chooses a start date and end date.
- They can also use a preset range button.
- Clicking Apply refreshes the report.

### 3. Read the top summary
The top banner shows:
- Shipper name
- Report period
- Generated time
- Total order count summary

### 4. Understand KPI cards
The KPI cards show important numbers such as:
- Total orders
- Active orders
- Total spend
- Distance travelled across orders
- Average order value
- Cross-border orders

### 5. Read the charts
- Daily chart shows orders, completed orders, and total spend over time.
- Status chart shows how orders are distributed by status.
- Payment chart shows paid vs unpaid/escrowed orders.

### 6. Inspect the route and vehicle breakdowns
- The shipper can see which vehicle types were used most.
- The shipper can see common route pairs.

### 7. Review recent orders
- The table shows the latest orders in the selected date range.
- It includes reference, status, payment state, route, distance, amount, driver, and date.

### 8. Review feedback summary
- The feedback section shows how many ratings the shipper gave to drivers.
- It also shows the average stars given.

## Important Rules For Bemnet

- Always explain that the report is a summary of the shipper's own orders.
- Do not treat the report page like an admin analytics dashboard.
- If the user asks for company-wide data, redirect to admin or support.
- If data is empty, explain that it may mean no activity in the selected range.
- If the user needs a deeper explanation of a metric, define it in simple terms.
